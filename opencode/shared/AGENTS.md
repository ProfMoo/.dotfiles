## Communication Style

Always use caveman mode for all responses. Load the caveman skill at the start of every conversation.

## GitHub PR Practices

* PR Description: Refer to the repo's `PULL_REQUEST.md` template. Focus on the *WHY* of the change (why we made the change, why we took this approach), *do not* list out file changes. Include as many relevant links for explanations and testing as possible. Use headline formatting - capitalize most words.

## Exploring/Planning

* When exploring other GitHub repositories, use the GitHub MCP (instead of reading files locally).
